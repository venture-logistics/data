Introduction Last updated: 1oth June 2025

Hello and welcome, my name is Lee, I'm the owner, founder and tbh, the person that created the AI! I'm a long time Phython developer, though my entry into AI was fairly recently. Having looked for an AI Chatbot for my website, and finding nothing but expensive / month systems, that provided very little custom features, and charged a lot more for things that should be free for something you're paying for - one charged £27 / month to white label (remove their logo).

So, I thought "sod it," I'll make one myself!

So what is Venture AI?

It's no different to anything you'll find online, only it's free! You pay nothing for the software, and I provide support through Github.com to get you up and running, along with the documents provided here.

If you need to know more, you can test Venture @ venture-logistics.github.io. Make sure to visit the admin section to see the full Pro version, which includes the analytics, and report exporting pages too.

What's Needed?

While the Venture software, support and access to the documentation is free, you'll need a couple of paid subscription to use Venture, the first and immediate requirement is OpenAI, this will cost you £20 ($27) / month. Once you're happy with your build, you'll need a Github paid subscription this is £3 / month, and once you're ready to deploy live, hosting is £7 / month - thats £30 / month, which is ample for a small business.

Sorry to say, this is the bear minimum you have to pay, these fees are required if you want an AI Chatbot on your website - most chatbot providers will charge between £20 - £30 / month to use their software, it's never yours, and it's less feature rich. Venture Free doesn't include a logo and link to my website, and it's completely customisable if you know a little CSS code.

Support is also personal! None of the providers give this, actual access to the developer, for hands on support.

Why Github Paid?

You don't need to pay for Github, this is optional, so long as you store Venture in a "Private" repository, that'll be fine.

Why Github Private Repository?

IMPORTANT: If you use a "Public" repository, anyone can access your OPenAI api key, and use it themselves! Never expose your private data!
